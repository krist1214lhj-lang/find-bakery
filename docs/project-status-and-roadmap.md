# 빵길 프로젝트 현황과 개발·배포 계획

작성 기준일: 2026-06-23

## 1. 프로젝트 목표

빵길은 단순히 검색 결과가 많은 서비스가 아니라, 여러 출처와 확인 시점을 근거로 사용자가 믿고 방문할 수 있는 빵집 탐색 앱을 목표로 한다.

핵심 원칙:

- 외부 검색 결과를 곧바로 확정 정보로 게시하지 않는다.
- 주소, 좌표, 전화번호, 영업시간, 메뉴를 항목별로 검증한다.
- 모든 주요 정보에 출처와 확인일을 남긴다.
- 출처가 충돌하거나 오래된 정보는 불확실성을 숨기지 않는다.
- 관리자 승인, 변경 이력, 사용자 제보와 재검증 절차를 운영한다.

## 2. 현재 개발 단계

현재 앱은 로컬 환경에서 실제 Supabase, 카카오 로컬 API, 소상공인 상가정보 API가 연결된 기능형 MVP다.

핵심 사용자 흐름과 외부 후보 검증 구조는 작동하지만, 실제 운영 데이터와 관리자 인증, 지도, 원격 배포 기반이 아직 부족하므로 공개 서비스 단계는 아니다.

## 3. 현재 구현 현황

### 사용자 기능

- 모바일 우선 홈 화면
- 지역, 빵집, 메뉴 검색
- 검증된 Supabase 빵집과 카카오 장소 후보 동시 검색
- 현재 위치 주변 검색과 위치 권한 거부 대안
- 빵 종류와 지역 필터
- 빵집 상세 정보
- 영업 상태와 임시 일정 계산
- 출처, 확인일, 검증 등급 표시
- 카카오맵 길찾기와 전화 연결
- 기기 기반 빵집 저장과 저장 해제
- 정보 오류·폐점·이전·메뉴 가격 제보

### 외부 데이터와 검증

- 카카오 로컬 API 실제 장소 검색
- 외부 응답 런타임 검증과 내부 모델 변환
- 외부 후보와 검증된 내부 데이터 분리
- 서버 서명 토큰을 사용한 후보 위조 방지
- 사용자가 선택한 후보만 검수 대기열에 저장
- 기존 지점과 상호·주소·전화·좌표 중복 비교
- 소상공인 상가정보 API 교차 확인
- 제과 관련 업종 필터
- 상호·주소·좌표 기반 일치 점수 계산
- 교차 확인 근거 영구 저장

### 관리자 기능

- 사용자 제보 검수
- 승인, 반려, 보류, 중복 처리
- 외부 장소 후보 검수
- 기존 지점 병합
- 검수 사유와 감사 이력 보존
- 승인 시 브랜드, 지점, 출처, 검증 기록 원자적 생성
- 공식 홈페이지·SNS·전화·현장 확인 등록
- 현재 저장값에 대한 항목별 A등급과 재검토 기한 생성
- 공식 계정·출처·검증·감사 이력 원자적 생성

### 신뢰 등급

- A: 최근 공식 채널 또는 매장 직접 확인
- B: 카카오와 공공데이터 등 독립된 출처가 일치
- C: 유효한 단일 출처 또는 일부 항목만 확인
- D: 사용자 제보, 출처 충돌 또는 재검토 필요

현재 구현된 자동 흐름:

```text
카카오 후보 발견
→ 공공 상가정보 교차 확인
→ 상호·주소·좌표 일치 점수 계산
→ 관리자 검수
→ 승인 시 정식 빵집 등록
→ 출처·확인일·감사 이력 생성
→ 강한 교차 확인 항목은 B등급
```

실제 통합 검증에서는 카카오 `퍼먼트`와 공공데이터 `퍼먼트알티드`가 상호 유사, 주소 일치, 좌표 1m 이내로 90점을 받았다. 승인 후 주소·좌표 B등급과 카카오·공공데이터 두 출처가 생성됨을 확인했다.

## 5. 현재 제한과 배포 차단 요소

### 데이터

- 공개 데이터는 실제 운영 데이터가 아니라 개발 시드 3곳 중심이다.
- 빵집 이미지와 이미지 사용 권리가 준비되지 않았다.
- SNS 공지 자동 수집과 임시 일정 입력 도구가 없다.
- 영업시간은 복수 구간과 자정 초과 영업을 완전히 지원하지 않는다.

### 사용자 경험

- 지도 화면과 지도·목록 동기화가 없다.
- 영업 중, 주차, 좌석, 포장, 검증 등급 필터가 완성되지 않았다.
- 로그인과 계정 간 저장 동기화가 없다.
- 항목별 전체 출처와 변경 이력 화면이 없다.

### 운영과 보안

- 현재 앱과 Supabase는 로컬 HTTP 환경이다.
- 원격 Supabase 테스트 프로젝트가 없다.
- 인증된 검수자·관리자 로그인이 없다.
- 공개 API의 지속형 속도 제한이 없다.
- 운영 모니터링, 알림, 자동 백업과 복구 검증이 없다.
- 자동 브라우저 회귀 테스트와 접근성 자동 검사가 없다.

이 항목들이 해결되기 전에는 공개 운영 배포를 완료 상태로 보지 않는다.

구현 항목:

- 영업시간·휴무 30일 재검토

- 지도만 사용하지 않아도 같은 정보를 목록에서 확인 가능

- 외부 API 장애 시 마지막 검증 데이터 표시
