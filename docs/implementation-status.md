# 구현 현황

업데이트: 2026-06-18

## 현재 단계

정적 시드 데이터를 이용한 상호작용형 프런트엔드 프로토타입.

## 구현됨

- Next.js 16, React 19, TypeScript 기반
- 모바일 우선 반응형 레이아웃
- 홈의 편집형 영상 히어로, 검색 진입, 빵 종류, 지역, 최근 확인 목록
- 빵지순례 출발표·검증 스탬프 모티프와 모션 감소 환경의 영상 대체 화면
- 지역·카테고리·검색어 기반 목록 탐색
- 빵집 상세의 영업 상태, 검증 배지, 메뉴 가격, 편의 정보, 길찾기
- 오래된 C 등급 정보에서 `영업 중`을 단정하지 않는 규칙
- 검색·검증 신선도·영업 상태 단위 테스트
- 빈 검색 결과와 404 화면
- 공식 원문 외부 링크
- 로컬 저장·해제와 새로고침 후 유지
- 저장 목록의 즉시 동기화와 빈 상태 복귀
- 정보 수정 제보 입력 검증과 로컬 접수 번호 발급
- 임시휴무·특별영업·변경 영업시간 우선순위 계산
- 문서 참조·린트·타입·테스트·빌드 통합 검증 명령
- Supabase 초기 스키마, 시드, RLS 정책과 검수 RPC
- API 우선·로컬 폴백 제보 저장 경계
- 로컬 관리자 검수 큐와 승인·반려·보류·중복 상태 전이
- 배포 환경 변수 검증과 스키마 계약 검사
- Docker 실행 이미지의 `public` 정적 자산 포함

## 알려진 제한

- 현재 빵집은 UI와 로직 검증을 위한 가상 시드 데이터다.
- 기본 영업시간은 아직 단일 일일 영업 구간만 사용한다.
- 저장은 기기 로컬 저장소 기반이며 계정 간 동기화되지 않는다.
- 제보는 관리자 서버 연결 전까지 해당 기기에만 보관된다.
- Docker가 없어 마이그레이션의 실제 PostgreSQL 적용과 RLS 런타임 검증은 아직 수행하지 못했다.
- `database.types.ts`는 생성 형식 호환 스냅샷이며 테스트 프로젝트 연결 후 CLI로 재생성해야 한다.
- 공개 제보 API의 운영용 지속형 속도 제한은 아직 연결하지 않았다.
- `npm run verify:deploy-env`는 현재 Supabase·배포 환경 변수가 없어 의도대로 실패하며 배포를 차단한다.
- 작업명 `빵길`은 공개 검색 1차 확인만 했으며 출시 전 상표·스토어 정식 확인이 필요하다.
